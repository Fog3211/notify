# 技术方案：每日 AI · 美股 · 财经资讯自动化简报（自建路线）

本文综合三份方案（本项目已搭的全代码自建版、一份「n8n + RSSHub 半自建」、一份
「免费积木 + GitHub Actions」），给出**以自建为主干**的目标架构与落地路线。
读者按模块 / 数据流理解即可，无需打开代码。

> ## 最终决策（已锁定）
>
> - **技术栈：Python**。决定因素是「在开源应用上二开」最顺手的底座几乎都是 Python；
>   去掉 A 股后 TS 也没有形成优势（详见对话纪要）。
> - **范围：只做美股，不做 A 股**。美股行情走免费 HTTP 接口，无需 A 股专属库。
> - **成本：尽量白嫖**。运行挂 GitHub Actions（零服务器、$0）；LLM 用免费档
>   （Gemini / Groq / Cloudflare Workers AI）或低价 DeepSeek。
> - **复用策略：吸收而非整仓 fork**。把 daily_stock_analysis 的「yfinance 行情 +
>   多模型 + Actions 部署」与 TrendRadar 的「多源聚合 + 多渠道推送 + LiteLLM」等
>   成熟做法**吸收进本项目**，自补两者都缺的**盘中暴涨暴跌异动监控**；不把带 A 股 /
>   港股 / 看板的大仓库整体搬进来，保持精简可控。
> - **关注领域：** AI / 大模型、美股科技、存储、CPO（光模块）、半导体、宏观财经。

## 1. 三方方案速览与取舍

| 方案来源 | 形态 | 优点 | 局限 |
|---|---|---|---|
| 本项目（已搭） | 全代码 Python + Docker | 逻辑在代码里、可测试可版本管理、彻底免费、灵活 | 需写代码；当前只做了新闻，未做行情与异动 |
| 半自建（n8n + RSSHub） | 低代码工作流 + 自建 RSS 中转 | 可视化拖拽、RSSHub 能把无 RSS 的站点转成源 | 复杂逻辑（去重/多 provider/异动）塞进 Function 节点难维护 |
| 免费积木（GitHub Actions） | 脚本 + 云端 cron | 零服务器成本、上手快、数据源/推送选项全 | 仍是写代码；偏「拼起来能跑」，缺工程结构 |

**结论：走自建（全代码）路线**，因为需求里有去重持久化、多 provider 切换、异动阈值
判断等真实逻辑，放代码里最易维护。同时**吸收另两份的长处**：

- 用 **RSSHub** 作为补充数据源，覆盖无原生 RSS 的财经站点（雪球、新浪财经、华尔街见闻、SEC 公告等）。
- 用 **yfinance** 免 key 拉美股**行情/价格**，补上「只做新闻、没做行情」的缺口。
- 新增**暴涨暴跌异动监控**与**盘中速报**这一条高频链路（原始需求明确要）。
- **n8n / GitHub Actions 仅作为可选触发器**，不承载业务逻辑；与本项目「触发器无关」的设计天然兼容。

## 2. 目标系统：两条链路、四类领域

系统按**两种节奏**运行，覆盖 AI / 大模型、美股科技、存储、CPO（光模块/共封装光学）、
半导体、宏观财经（A 股为可选扩展）。

```
盘中高频链路（每小时 · 美股交易时段）
  拉行情快照 + 大事件新闻 → 计算涨跌幅/成交量 → 阈值判定异动
    → 命中则 AI 速评 → 推送「异动/大事件速报」

每日定时链路（开盘前 / 收盘后）
  汇总当日新闻 + 行情 → 窗口过滤 + 去重 → 按主题分组
    → AI 结构化分析 → 推送「每日简报」
```

两条链路共用同一套采集、AI、推送、存储能力，只是触发节奏与分析提示词不同。

## 4. 关键设计决策

| 决策 | 取舍理由 |
|---|---|
| 一次运行即幂等 | 任何触发器跑一遍都安全，靠去重保证不重复推送 |
| 触发器无关 | 业务逻辑收敛在「跑一次」的命令里，Docker/cron/n8n/GitHub Actions 都能驱动 |
| 行情用免费源 + 容延迟 | 免费行情普遍约 15 分钟延迟，仅用于看趋势/找异动，**不做高频或秒级交易判断** |
| LLM provider 可切换 | 配置选 provider/model，按场景在 Claude/OpenAI/DeepSeek/通义间切换 |
| 单源容错 | 任一数据源失败只记录并跳过，不拖垮整条链路 |
| 异动按冷却期去重 | 同一标的同方向异动在冷却窗口内只推一次，避免刷屏 |
| AI 只做信息加工 | 仅摘要、归因、异动提醒；不预测涨跌、不给买卖点（见第 7 节） |

## 5. 异动检测设计（新增核心能力）

判定输入为标的的**最新价、前收价、当日成交量、历史均量**，按以下口径触发：

- 日内涨跌幅 = (最新价 − 前收价) / 前收价；`|日内涨跌幅| ≥ 日阈值`（默认 5%）触发。
- 小时涨跌幅 = (最新价 − 上一快照价) / 上一快照价；`|小时涨跌幅| ≥ 小时阈值`（默认 3%）触发。
- 量能异常：当日成交量 ≥ 历史均量 × 倍数阈值（默认 3×）触发。
- 命中后关联同标的近段新闻，交 AI 做一句话归因，组成速报推送。
- 冷却期内（默认 4 小时）同标的同方向不重复推送。

阈值、冷却期、关注标的均来自配置，便于按市场波动性调参。

## 6. 相对现状的增量（落地路线图）

按「功能 + 模块」描述要做的事，分阶段推进：

**阶段一 · 行情与异动（本次实现，补齐原始需求的核心缺口）**
- 行情采集模块：免 key 拉美股报价（Yahoo chart 接口为主、Stooq CSV 兜底），落地为统一的行情数据结构。
- 行情快照存储：每次拉取写入基线，供下次小时级对比。
- 异动检测模块：实现第 5 节口径，产出异动告警对象，按冷却期去重。
- 异动速报渲染：纯规则即可推送（无需 LLM）；配置了 LLM 时叠加一句话 AI 归因。

**阶段二 · 多频率调度（本次实现）**
- 调度扩展为「盘中每小时」+「每日定时」两类任务，盘中任务仅在美股交易时段运行。
- 部署优先 GitHub Actions（两个 cron 工作流，零服务器、$0）。

**阶段三 · 数据源扩充（按需）**
- 接入 RSSHub 源（雪球/新浪/华尔街见闻/SEC 等）。
- 可选接入新闻 API（NewsAPI/TianAPI）。**A 股不做。**

**阶段四 · 推送渠道扩充（按需）**
- 企业微信机器人、Bark、ntfy、Telegram Bot 作为可选渠道接入。

**阶段五 · 部署固化**
- Docker 常驻为主；提供宿主 cron / GitHub Actions 模板作为备选触发器。

## 7. 约束与免责

- **非投资建议**：本系统只做信息采集、摘要与异动提醒，**不预测涨跌、不给买卖点**，
  产出仅供参考，决策自负。
- **行情有延迟**：免费数据源约 15 分钟延迟，不适用于高频/秒级交易。
- **不做交易执行**：系统不接券商、不下单，只读不写。
- **数据源稳定性**：第三方源可能限流或失效，单源失败不影响整体，但需定期校验源可用性。

## 8. 参考：触发方式对照

| 触发方式 | 适用 | 说明 |
|---|---|---|
| Docker 常驻 + 内置调度 | 有 NAS / 长期开机的机器 | 推荐主用，去重库随卷持久化 |
| 宿主 cron / launchd | 本地机器定时唤起 | 跑一次即退，省资源 |
| GitHub Actions | 不想自备服务器 | 公开仓库免费 cron，密钥放 repo secrets |
| n8n Schedule 节点 | 已有 n8n | 仅作定时触发器去调用本项目命令 |
